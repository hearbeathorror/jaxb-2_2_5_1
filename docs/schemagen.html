Launching schemagen

The schema generator can be launched using the appropriate schemagen shell script in the bin directory for your platform. For this Early Access release, we are only providing a basic shell script for evaluation purposes. Future releases will contain more robust schema generation tools.

The current schema generator processes Java source files only. Future versions of the tool may also be capable of processing compiled class files.

We also provide an Ant task to run the schema generator - see the instructions for using schemagen with Ant.

For Solaris/Linux

      % $JAXB_HOME/bin/schemagen.sh Foo.java Bar.java ...
      Note: Writing schema1.xsd

For WindowsNT/2000/XP

      > %JAXB_HOME%\bin\schemagen.bat Foo.java Bar.java ...
      Note: Writing schema1.xsd

If your java sources reference other classes, they must be accessable on your system CLASSPATH environment variable or else you will see errors when generating your schema.

Command Line Options

Usage: schemagen [-help ...] <java files>

Options:
  -d <path>                  Specify where to place processor and javac generated class files
NO OTHER APT COMMANDLINE OPTIONS ARE SUPPORTED AT THIS TIME.

Generated Resource Files

The current schema generator simply creates a schema file for each namespace referenced in your Java classes. There is no way to control the name of the generated schema files at this time. Use the schema generator ant task for that purpose.
